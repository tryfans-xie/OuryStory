# 🌌 「心动引力波 OurStory」MVP 产品引擎与功能矩阵路线图

## 🎯 一、 核心引擎：MVP 价值主张与北极星指标

**🚀 MVP 核心价值主张（Value Proposition）**
打造 Z世代专属的**赛博情感自留地**。以多模态 AI 驱动沉浸式时间轴，为情侣/夫妻提供高私密、高颜值的数字记忆空间，将碎片化的日常心动转化为可触摸的情感资产。

**📊 MVP 北极星指标与验证目标**

1. **价值验证**：测试用户为“专属私密情绪价值空间”付费的意愿（订阅转化率）。
2. **体验验证**：跑通“四维时间轴 + AI 多模态相册 + 情绪胶囊”的核心交互闭环，验证 Aha Moment（顿悟时刻）。
3. **安全验证**：构建基于零知识证明与端到端加密的隐私护城河，确保数据绝对安全。

采用 **“Web 先行，桌面壳跟进”** 的策略，确保快速验证市场。

#### ***\*Phase 1: 核心基座与视觉原型 \****

- **目标**：跑通 Next.js + Tailwind + Shadcn 的基础框架，确立“毛玻璃+暗黑赛博”的 UI 规范。
- **关键交付**：
  - 完成跨端扫码登录授权链路。
  - 实现基于 IndexedDB 的本地缓存机制。
  - 完成“黑洞级”拖拽上传组件及进度条动效。

#### ***\*Phase 2: 时空引擎与 AIGC 工坊 \****

- **目标**：攻克 PC 端最核心的视觉与交互壁垒。
- **关键交付**：
  - **Three.js 3D 星系时间轴**首版上线（支持滚轮缩放与节点点击）。
  - 接入大模型 API，跑通“AI 一键图文混排”与“AI 废片初筛”后端链路。
  - 实现类似 Figma 的无限画布基础拖拽与吸附功能。

#### ***\*Phase 3: Tauri 桌面化与极客体验 \****

- **目标**：将 Web 应用封装为原生桌面体验，释放本地算力。
- **关键交付**：
  - 引入 Tauri 2.0，打包 Windows/macOS 安装包。
  - 实现本地文件夹监听自动备份功能。
  - 开发桌面悬浮挂件（情话/纪念日倒计时）。
  - 完善全局快捷键矩阵。

#### ***\*Phase 4: 闭环测试与内测发布****

- **目标**：性能调优，准备种子用户内测。
- **关键交付**：
  - 使用 Lighthouse 和 React Profiler 进行极致的渲染性能优化（确保 3D 时间轴在普通轻薄本上也能跑满 60fps）。
  - 完成端到端加密（E2EE）的全链路安全审计。
  - 发布“赛博招募令”，邀请 100 对极客/设计圈情侣进行封闭内测。

## ⚠️ 三、 极客避坑指南（Tech Stack & Architecture Warnings）

> **💡 架构师箴言：不要在 MVP 阶段埋下技术债务的种子。**

1. **🚫 抛弃 SQLite，拥抱 PostgreSQL**
   - **痛点**：智能相册涉及 AI Worker 异步写入与用户高频请求的并发，SQLite 的表级锁会导致严重的性能瓶颈。
   - **解法**：Day 1 就使用 PostgreSQL，利用其强大的并发处理能力和 JSONB 支持。
2. **🚫 拒绝前端 AI 推理，算力后置**
   - **痛点**：在浏览器端跑 WASM 人脸/物体识别模型，不仅推理极慢，还会导致手机发热降频，体验灾难。
   - **解法**：AI 推理必须全部在后端 GPU Worker 或云函数中异步完成，前端只做结果渲染。
3. **🚫 警惕 EXIF 时区陷阱**
   - **痛点**：跨国旅行或不同手机系统的时区设置混乱，会导致时间轴错乱。
   - **解法**：后端统一将时间戳转换为 **UTC 存储**，前端根据用户当前设备的 Local Timezone 动态转换渲染。
4. **🚫 向量数据库不要过早优化**
   - **痛点**：MVP 阶段引入独立的 Milvus 或 Pinecone 会增加运维复杂度。
   - **解法**：初期直接使用 PostgreSQL 的 **`pgvector` 插件** 进行语义检索，当向量数据突破 50 万级时再考虑迁移至专业向量库。
5. **🚫 API 版本控制是底线**
   - **痛点**：前期不规划，后期多端（iOS/Android/Web/小程序）兼容时面临重构灾难。
   - **解法**：从第一行代码开始，所有接口强制加上 `/api/v1/` 前缀，为未来的无感升级留出余地。
